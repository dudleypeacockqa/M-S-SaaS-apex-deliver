# FinanceFlo.ai Brand Identity Guide

**Version**: 1.0  
**Date**: November 24, 2025  
**Purpose**: Comprehensive brand guidelines for FinanceFlo.ai website redesign

---

## Executive Summary

FinanceFlo.ai is a finance automation platform that combines AI with ERP integration to transform finance operations for UK mid-market businesses. The brand identity reflects professionalism, technological innovation, and trustworthiness through a carefully crafted visual system centered on deep navy blues and vibrant emerald greens.

### Brand Pillars

1. **Precision Intelligence** – Communicate advanced automation with crisp geometry, high-contrast typography, and data-forward imagery.
2. **Operational Confidence** – Reinforce trust through consistent navy foundations, rigorous spacing, and clear hierarchy.
3. **Momentum & Growth** – Use emerald gradients, upward motion cues, and ROI statistics to spotlight measurable outcomes.
4. **Human Partnership** – Balance the tech narrative with collaborative photography, approachable copy, and inclusive visuals.

Each visual decision in this guide should map back to at least one pillar so teams can quickly validate design choices.

## Logo System

### Primary Logo

The FinanceFlo.ai logo consists of two elements that work together to create a cohesive brand mark.

**Logo Components**:
- **Icon**: Circular "F" symbol in dark navy blue
- **Wordmark**: "FinanceFlo" in professional sans-serif + ".AI" in bright green
- **Format**: Horizontal lockup (icon left, text right)

**Logo Specifications**:
| Element | Typeface | Color | Weight |
|---------|----------|-------|--------|
| Icon | Custom symbol | Navy (#081629) | Bold |
| "FinanceFlo" | Sans-serif | Navy (#081629) | Bold |
| ".AI" | Sans-serif | Emerald (#10b981) | Bold |

**Clear Space**: Maintain minimum clear space equal to the height of the "F" icon on all sides.

**Minimum Size**: 
- Digital: 120px width
- Print: 30mm width

### Logo Variations

**Primary**: Full color on light backgrounds  
**Reverse**: White version on dark backgrounds  
**Monochrome**: Navy version when color is not available  
**Icon Only**: Circular "F" for small applications (favicons, app icons)

### Logo Usage Guidelines

**Do**:
- Use the provided logo files without modification
- Maintain proper clear space
- Scale proportionally
- Use on appropriate backgrounds

**Don't**:
- Stretch, skew, or rotate the logo
- Change colors or add effects
- Separate icon from wordmark in primary applications
- Place on busy or low-contrast backgrounds

### File Formats & Placement

- **File Types**: Use provided SVG assets for digital and EPS for print to preserve crisp edges.
- **Backgrounds**: Prefer solid navy or white; when placed on imagery, apply a 40% navy overlay to maintain contrast.
- **Alignment**: Left-align the horizontal lockup for most layouts, centering only within hero, footer, or standalone callouts.
- **Safe Zones**: When pairing with partner marks, insert a divider line or at least 24px padding to avoid brand collisions.

## Typography

### Font Families

**Primary Typeface**: Inter (sans-serif)  
**Fallback Stack**: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif

**Why Inter**: 
- Excellent readability at all sizes
- Professional and modern appearance
- Wide range of weights
- Optimized for digital screens
- Open-source and freely available

### Type Scale

| Style | Size | Weight | Line Height | Letter Spacing | Usage |
|-------|------|--------|-------------|----------------|-------|
| **Display** | 72px | 700 Bold | 1.1 | -0.02em | Hero headlines |
| **H1** | 48px | 700 Bold | 1.2 | -0.01em | Page titles |
| **H2** | 36px | 600 Semibold | 1.3 | -0.01em | Section headings |
| **H3** | 30px | 600 Semibold | 1.4 | 0 | Subsection headings |
| **H4** | 24px | 600 Semibold | 1.4 | 0 | Card titles |
| **H5** | 20px | 600 Semibold | 1.5 | 0 | Small headings |
| **H6** | 18px | 600 Semibold | 1.5 | 0 | Overlines |
| **Body Large** | 18px | 400 Regular | 1.7 | 0 | Lead paragraphs |
| **Body** | 16px | 400 Regular | 1.6 | 0 | Body text |
| **Body Small** | 14px | 400 Regular | 1.6 | 0 | Secondary text |
| **Caption** | 12px | 400 Regular | 1.5 | 0.01em | Labels, captions |
| **Button** | 16px | 600 Semibold | 1 | 0.01em | Button text |

### Typography Guidelines

**Headings**:
- Use bold weights (600-700) for emphasis
- Maintain clear hierarchy with size and weight
- Apply tighter line-height for large headings
- Use negative letter-spacing for display text

**Body Text**:
- Use 16px as base size for optimal readability
- Maintain 1.6 line-height for comfortable reading
- Keep line length between 60-80 characters
- Use Gray 700 (#374151) for primary body text

### Type Pairings

- **Hero Stack**: Display (72px) headline + Body Large (18px) subcopy; apply emerald gradient only to keywords.
- **Section Head**: H2 (36px) paired with Body (16px) and 24px spacing for scanability.
- **Cards**: H4 (24px) for titles with Body Small (14px) for supporting text to keep layouts compact.
- **Legal / Meta**: Caption (12px) uppercase with 0.08em letter-spacing for labels, overlines, and footer copy.
- **Data & Metrics**: Enable `font-feature-settings: 'tnum' 1` for tables, dashboards, and pricing columns.

**Emphasis**:
- **Bold** (600 weight) for strong emphasis
- *Italic* for subtle emphasis or quotes
- Emerald Primary (#10b981) for links and highlights
- UPPERCASE with increased letter-spacing for labels

## Spacing System

### Base Unit

**Base**: 4px (0.25rem)

All spacing follows an 8-point grid system for consistency and rhythm.

### Spacing Scale

| Token | Value | Pixels | Usage |
|-------|-------|--------|-------|
| xs | 0.5rem | 8px | Tight spacing, icon gaps |
| sm | 0.75rem | 12px | Small gaps, button padding |
| md | 1rem | 16px | Default spacing |
| lg | 1.5rem | 24px | Section padding |
| xl | 2rem | 32px | Large gaps |
| 2xl | 3rem | 48px | Section margins |
| 3xl | 4rem | 64px | Major sections |
| 4xl | 6rem | 96px | Hero sections |
| 5xl | 8rem | 128px | Extra large spacing |

### Layout Guidelines

**Container Width**: 1280px maximum (80rem)  
**Content Width**: 768px for reading content (48rem)  
**Section Padding**: 4rem vertical (64px)  
**Card Padding**: 1.5rem (24px)  
**Button Padding**: 0.75rem horizontal, 0.5rem vertical (12px × 8px)

### Spacing Guidelines

- **Do** snap sections to consistent 8px increments and align baselines across breakpoints.
- **Do** reserve 3xl (64px) and above for hero, testimonials, or CTA breathing room.
- **Don't** mix different padding values within the same component tier (cards, buttons, inputs).
- **Don't** place two high-density blocks back-to-back without a divider, background shift, or 2xl spacing buffer.

## Animation & Motion

### Animation Principles

**Purpose**: Enhance user experience, not distract  
**Duration**: 200-300ms for micro-interactions, 400-600ms for page transitions  
**Easing**: ease-in-out for most animations, ease-out for entrances

### Common Animations

**Button Hover**:
```css
transition: all 200ms ease-in-out;
transform: translateY(-2px);
box-shadow: 0 4px 6px rgba(0,0,0,0.1);
```

**Card Hover**:
```css
transition: all 300ms ease-in-out;
transform: translateY(-4px);
box-shadow: 0 10px 20px rgba(0,0,0,0.1);
```

**Fade In**:
```css
animation: fadeIn 400ms ease-out;
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(20px); }
  to { opacity: 1; transform: translateY(0); }
}
```

**Slide In**:
```css
animation: slideIn 600ms ease-out;
@keyframes slideIn {
  from { opacity: 0; transform: translateX(-40px); }
  to { opacity: 1; transform: translateX(0); }
}
```

## Voice & Tone

### Brand Voice

**Professional**: Knowledgeable and trustworthy  
**Confident**: Clear about capabilities and results  
**Approachable**: Friendly without being casual  
**Results-Oriented**: Focus on outcomes and metrics  
**UK-Focused**: Tailored for UK mid-market businesses

### Writing Guidelines

**Do**:
- Use specific metrics and data points
- Write in active voice
- Keep sentences concise and clear
- Use UK English spelling
- Address the reader directly ("you", "your")
- Emphasize benefits over features

**Don't**:
- Use jargon without explanation
- Make unsupported claims
- Write in passive voice
- Use American English spelling
- Be overly technical or academic
- Focus solely on features

### Example Copy

**Good**: "Transform your finance operations with AI+ERP. Join 450+ UK businesses achieving 66% cost reduction and 500% ROI boost."

**Bad**: "Our solution leverages cutting-edge artificial intelligence and enterprise resource planning integration to facilitate operational transformation."

### Voice Checklist

- Lead with a metric or benefit within the first sentence of each section.
- Pair every claim with proof (statistic, testimonial, certification).
- Reinforce the UK focus via spelling, currency, and regulatory references when relevant.
- Close copy blocks with a clear action cue (CTA, link, or “next step” language).

### Design Token Quick Reference

- **Color tokens** map to Tailwind keys defined above (e.g., `text-navy-primary`, `bg-emerald-primary`) and should be the single source for backgrounds, text, and borders.
- **Typography tokens** (`display`, `h1`-`h6`, `body`, `caption`) correspond to CSS custom properties in the `:root` block—avoid ad-hoc font sizing outside this scale.
- **Spacing tokens** follow the 4px base; use Tailwind spacing utilities (`gap-6`, `px-8`) as shorthand for the table in this guide.
- **Shadow tokens** (`shadow-sm`, `shadow-md`, `shadow-lg`) are standardized for buttons, cards, and modals—do not introduce unique drop shadows.
